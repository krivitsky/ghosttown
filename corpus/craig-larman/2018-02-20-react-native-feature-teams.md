when introducing the concept of feature teams, will often hear variations
of "our ios devs can't learn android", etc ad nauseum. leaving aside the
issue of learning, there are also simplifying old-ish (and costly)
technical solutions such as xamarin for cross-platform dev

but now react native <https://facebook.github.io/react-native/> (FOSS) is
taking off, and worth being aware of, as it provides yet another technical
solution to reduce tech stack heterogeneity. as y'all probably know,
react.js <https://reactjs.org/> has become (since 2011) a major choice in
the web ui world. in 2015 facebook (the creators of react.js) also released
react native, that allows devs to use the same react ui components & FW
(and relay <https://facebook.github.io/relay/>), and *dev in only js and
react (no java or swift required)*, for ios & android. the facebook app
uses it (and instagram, etc)... and facebook is working towards seamless
ability to merge react native and react.js so that android/ios/web ui dev
is all the same.

branching back to org design, when have shared this with questioners, have
noticed on many occasions that the questioner is kinda unhappy to hear
that, almost as though, "damn, there must be *some* reason we have to cling
to component teams and maintain the status quo" ;)
