a while back we had a "react native" thread, and steve alexander shared positive experiences with flutter for cross-mobile platform dev

sorta repeating the opening motivation from that thread:
when introducing the concept of feature teams, will often hear variations of "our ios devs can't learn android", etc ad nauseum, driving the continuation of component teams

react native is one technical solution, with same code for ios & android, and meaningful cross-over to react.js for web, though more code variation for the web version

google's flutter (ios/android) has been growing in popularity, and they have now released a beta of flutter for web, with apparently same code as for ios/android. to state the obvious: same code, 3 I/O channels. the web version is not ready for production use, but given google's focus & talent, one can expect quick progress
